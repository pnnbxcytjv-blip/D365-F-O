# 1.3 Dynamics 365 Finance

{% hint style="info" %}
财务模块快速入门
{% endhint %}

### 概述

Microsoft Dynamics 365 是一个由基于 Azure 的技术提供支持的集中式平台，可集成业务功能。 它可以无缝管理财务、供应链管理、人力资源、商务和项目运营的业务流程。 协作环境有助于实现实时数据可用性和跨应用程序的数据交换。 Finance 是连接到所有财务和运营的应用程序，可确保集中记录财务影响并按照法规合规性进行报告。

Finance 支持所有主要的财务管理功能，其中包括：

* 总帐。
* 现金和银行管理。
* 应付帐款。
* 应收帐款。
* 固定资产。
* 预算编制。

Finance 专为满足组织的多公司多币种业务的需求而设计，可实时监控公司的绩效。 Finance 可预测未来结果，并做出数据驱动型决策，以促进组织的发展，无论其规模和运营所在的行业如何。 下图重点介绍了 Finance 的一些功能。

### &#x20;总帐

通过总帐模块，您可以定义和管理法人的财务记录。 总帐是借方和贷方条目的登记簿。 这些条目使用会计科目表中列出的科目进行分类。

**用例：**&#x67D0;个组织正在实施 Finance 以实现其财务和会计系统的数字化。 总帐是组织设计的第一个模块。 您需要配置会计年度，然后配置会计科目表和主科目。 然后，您需要定义维度和维度集，然后是会计结构。 最后，您需要设置分类帐，在其中链接会计年度、会计科目表和会计结构。 您还需要在分类帐设置中定义默认货币和默认汇率。 您的所有会计交易记录均遵循您在分类帐设置中实现的配置。

### 应收帐款 <a href="#accounts-receivable" id="accounts-receivable"></a>

应收帐款 (AR) 是在产品或服务交付且开票后应向客户收取的余额。 所有应收帐款的总值在资产负债表上作为流动资产列出。 使用 AR 跟踪客户发票和收入付款。 您可以创建基于销售订单或装箱单的客户发票。 如果发票与销售订单无关，则可以创建普通发票。 您还可以使用包括汇票、现金、支票、信用卡和电子付款在内的几种付款类型接收付款。

**用例：**&#x60A8;是一名应收帐款经理。 您需要对分类帐簿中的所有应收帐款交易记录进行核算，例如客户发票和供应商退货。 应收帐款模块可对带有付款交易记录的所有应收帐款进行结算。 该模块支持不同的付款方式、付款期限、现金折扣规则、付款日和日历。

### 应付帐款 <a href="#accounts-payable" id="accounts-payable"></a>

应付帐款 (AP) 是公司因货物或服务已接收但尚未付款而拖欠供应商的金额。 所有应付帐款的总值在资产负债表上作为负债列出。 您可以手动输入供应商发票或通过集成服务以电子方式导入这些发票。 输入或接收发票后，您可以使用发票审批日记帐对其进行审查和审核。 该系统支持使用发票匹配、供应商发票政策和工作流来自动执行审查流程。 满足特定条件的发票将自动获得批准，其余发票将标记为待审查。

**用例：**&#x60A8;是一名应付帐款经理，需要对分类帐簿中的所有应付帐款交易记录进行核算，例如供应商发票和客户退货。 应付帐款模块可对带有您进行的付款交易记录的所有应付帐款进行结算。 该模块支持不同的付款方式，例如：

* 支票和电子付款
* 付款条件
* 现金折扣规则
* 付款日
* 日历。

### 成本核算 <a href="#cost-accounting" id="cost-accounting"></a>

成本核算是会计的一个分支，涉及记录、分类、分析和汇总企业产生成本的流程。 成本核算模块管理成本核算中使用的流程，包括分配基础管理、成本核算分类帐维护、成本条目、成本分类并了解成本行为。

**用例：**&#x60A8;是一家跨国公司的区域运营主管。 您负责四个国家/地区的运营，这四个国家/地区由四个法人分别代表。 您有一个成本中心来管理各个国家/地区的运营。 成本核算模块可帮助您对法人的日常运营进行核算。 该模块通过跟踪和衡量资源并研究其影响来帮助您决定如何有效地利用资源。 它还指导您进行成本控制。

### 预算编制 <a href="#budgeting" id="budgeting"></a>

本模块概述了预算编制功能组成部分、预算编制工具和报告功能。 Dynamics 365 中的预算编制模块分为三个主要部分：预算计划、预算条目和预算控制。 预算计划是确保在实施预算时满足组织策略、程序和要求的流程。 预算条目（即下一步骤），有助于在 Finance 中登记预算行项并使用总帐科目映射行项。 预算控制是根据登记的预算行验证 Finance 中的财务交易记录的可选步骤。

**用例：**&#x60A8;是一家组织的财务总监。 您使用 Finance 来确保长期战略计划和年度预算计划，以跟踪财务交易记录是否与组织层次结构保持一致。 您可以为组织的会计科目表中定义的主科目和维度配置预算分配。 您还可以根据既定预算在交易记录级别实施预算控制。 控制的级别取决于组织文化和组织的成熟度级别。

### 现金和银行管理 <a href="#cash-and-bank-management" id="cash-and-bank-management"></a>

您可以使用“现金和银行管理”模块来维护法人的银行帐户以及与这些银行帐户关联的财务票据。 这些票据包括存款单、支票、汇票和本票。 您还可以对银行对帐单进行对帐，以及在标准报表上打印银行数据。

**用例：**&#x60A8;是一名会计，负责维护组织银行帐户以及执行“现金和银行管理”模块内所有与银行相关的操作。 如果您需要将银行对帐单与分类帐簿进行对帐，可以将银行对帐单导入到 Finance，然后从银行交易记录中自动对其进行对帐。 您还可以在此模块中预测现金流量数据。 根据应付帐款和应收帐款数据以及付款条件，可以得出您组织的现金流量预测。

### 固定资产 <a href="#fixed-assets" id="fixed-assets"></a>

本模块提供对使用 Finance 固定资产的资源的访问权限。 固定资产是指由个人或组织拥有的有价物料。 这些物料包括建筑物、车辆、土地和设备。

**用例：**&#x60A8;的组织依靠各种固定资产来运营，其中包括财产、机械和库存等有形物料。 您是公司的一名会计，使用这些资产来生成组织的产品和服务。 组织拥有无形资产，包括版权、品牌和商誉，这些资产有助于提升组织的未来价值，有时甚至比有形资产更有价值。 在“固定资产”模块中，您可以维护资产的概况，例如采购详细信息、当前评估和折旧概况。

### 支出管理 <a href="#expense-management" id="expense-management"></a>

通过“支出管理”模块，您可以创建一个集成的工作流来存储付款方式信息，导入信用卡交易记录并跟踪员工的支出。 您还可以定义支出政策并自动报销差旅费。

**用例：**&#x516C;司有针对不同支出（例如差旅和食品）制定的规则和政策。 您是该公司的人力资源经理。 在“支出管理”模块中，根据组织层次结构定义所有这些政策。 例如，当员工记录支出时，他们需遵循已制定的政策。 还可以针对跨法人应用场景记录支出，在这种应用场景中，员工处理属于与其他法人关联的项目，并且需要将这些支出分配到特定项目。

### 监管功能 <a href="#regulatory-features" id="regulatory-features"></a>

财务和运营应用包含适用于不同国家/地区的监管功能。 此功能基于有效法人的主要地址启用。 此功能具有监管更新，以便支持新的或更改的特定于国家或特定于地区的立法。 作为修补程序 (HF) 或发布预览（预览）的一部分提供的监管更新。 它还可以提交和检查监管预警。&#x20;

### 资产租赁 <a href="#asset-leasing" id="asset-leasing"></a>

Finance 提供用于管理、跟踪和自动执行租赁资产财务交易记录的高级功能。 本模块处理租赁详细信息，生成日记帐条目并管理整个租赁生命周期，包括从初始确认到每月日记帐条目、减损和租赁终止。

**用例：**&#x8D35;公司计划扩大业务。 您计划租赁这些物料，而不是购买家具。 资产租赁流程包括租赁协议、租赁计算和租赁交易记录等交易记录。 这些交易记录根据定义的过帐规则记入分类帐簿。 借助 Finance，公司能够维护此流程以及各种租赁组成部分，例如付款计划、开始和结束日期、付款频率和租赁摊销。

### 订阅计费 <a href="#subscription-billing" id="subscription-billing"></a>

通过订阅计费，您可以使用计费计划来管理订阅收入和定期计费。 该功能支持定期合同计费、收入、支出延期和多元素收入分配。 您可以根据组织的需求一起或单独使用这些组件。

### 信用和收款管理 <a href="#credit-and-collection-management" id="credit-and-collection-management"></a>

通过客户信用管理，您可以根据您创建的信用规则，通过过帐流程管理信用额度并控制销售订单的流。 收款管理集中监督应收帐款收款数据。 “应收帐款”模块提供付款条件和计划，可定义收款生成流程。 它可以根据定义的计划生成收款提醒。 它还可以在有款项逾期未付时计算利息。
